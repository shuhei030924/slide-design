# HTMLスライド作成 (PPT 1枚)

PowerPointのデフォルト横スライドと同一サイズ(16:9、1280×720px)のスライド1枚を、単一のHTMLファイルとして作成する。

## 絶対ルール(これだけは必ず守る)

1. **サイズ**: `.slide` は `1280px × 720px` 固定 + `overflow: hidden`(はみ出し禁止)
2. **1枚で完結**: ページ番号・連番・「次ページへ続く」は入れない。収まらなければ内容を削る
3. **単一ファイル**: CSSは `<style>` にインライン。外部依存はGoogle Fontsのみ
4. **テンプレートから作る**: ゼロから書かず [template.html](./assets/template.html)(ライト)か [template-dark.html](./assets/template-dark.html)(ダーク)をコピーして改変する。近いパターンの実作例(`slides/slide-*.html`)があればそちらをベースにする方がさらに良い
5. **色を発明しない**: 配色は [デザインガイド](./references/design-guide.md) の6テーマから選び、`:root` トークンを丸ごとコピーする
6. **タイトルはメッセージ文**: 「〜の概要」のような名詞ラベルではなく「売上高は6年連続成長、128億円へ。」のように結論を言い切る
7. **出力先**: ワークスペース直下の `slides/` に `slide-<内容名>.html` で保存

## 手順(この順番で進める)

### 1. 内容の把握
依頼からタイトル(=伝えたい結論)・データ・図解の要否を整理する。不明点が多ければ質問する。

### 2. パターン選択 — [パターン選択ガイド](./references/pattern-guide.md) を読む
判定フローに従い35パターンから選ぶ。迷ったら②ビジネス型。
選んだパターンに実作例(`slides/slide-*.html`)があれば、**そのファイルを読んでベースにする**。

### 3. デザイン決定 — [デザインガイド](./references/design-guide.md) を読む
- 内容のトーンに合うカラーテーマ(A〜F)を1つ選び、`:root` をコピー
- スライドの4層構造(キッカー/タイトル/ボディ/フッター)・文字サイズ表・余白ルールに従う
- 「やってはいけない集」に該当しないか確認

### 4. 作成
- テンプレートまたは実作例をコピーして改変
- 図解が必要なら**先に [SVG図解作成ガイド](./references/svg-guide.md) を読んでから**インラインSVGで描く(座標は計算式をコメントに残す)
- ユーザー提供画像があれば相対パスで配置(下記「ユーザー提供画像」)

### 5. 検証(必須 — 目視せずに完了報告しない)
ヘッドレスブラウザでスクリーンショットを撮り、画像を確認する:

```powershell
# Windows(Chromeのパスは環境に合わせる)
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --headless=new --disable-gpu `
  --screenshot="<出力先>.png" --window-size=1280,720 --hide-scrollbars `
  --virtual-time-budget=10000 "file:///<スライドの絶対パス>.html"
```

確認する項目: ①文字のはみ出し・重なり ②SVG内テキストのずれ ③余白が潰れていないか。
問題があれば修正して**再スクリーンショット**する。ブラウザが使えない環境では、その旨を伝えてユーザーに目視確認を依頼する。

## レイアウトパターン(概要)

全35パターンの選び方は [パターン選択ガイド](./references/pattern-guide.md) を参照。主要どころ:

| パターン | 用途 | 実作例 |
|---|---|---|
| ビジネス型(デフォルト) | 要点説明・提案の骨子 | slide-business |
| コンセプト型 | 表紙・ビジョン | slide-refined |
| KPIデータ型 | 数値実績の強調 | slide-kpi |
| グラフ型 | 推移データ+考察 | slide-chart |
| 3カード型 | 並列3〜4項目の比較 | slide-cards |
| ベントーグリッド型 | 多種の情報を1枚で俯瞰 | slide-bento |
| 画像ヒーロー型 | 全面ビジュアル+オーバーレイ | slide-hero |
| フロー型 | 手順・ステップ | slide-flow |
| 対比型 | Before/After | slide-versus |

## SVG図解

図・アイコン・チャートはインラインSVGで自作する。
**SVGを描く前に必ず [SVG図解作成ガイド](./references/svg-guide.md) を読むこと。**
座標計算式・テキスト中央揃えの公式・棒/円/折れ線グラフの定型コード・セルフチェックリストが載っている。

最低限のルール(ガイド未読でも守ること):
- viewBoxの論理座標で描き、座標は先に計算してコメントに残す
- テキスト中央揃えは `text-anchor="middle"` + `y = 中心y + fontSize×0.35`(dominant-baselineは使わない)
- 日本語テキスト幅 = 全角文字数 × fontSize で見積り、ボックスに収まるか確認
- `<defs>` の id はHTML内で一意にする(複数SVGでの重複禁止)

## ユーザー提供画像

- スライドHTMLからの相対パスで参照(例: `../images/photo.png`)。画像の場所を必ず確認する
- `object-fit: cover` + 固定サイズのコンテナでトリミング配置
- 全面背景に使う場合は [slide-hero.html](../../../slides/slide-hero.html) の構造(bg-media層+overlay層)をベースにする。オーバーレイなしで文字を乗せない

## 会社ロゴ

- 配置場所: 右上(トップバーの右端)が基本。タイトルスライドでは中央上や左上も可
- コンテナは `height: 36〜44px; max-width: 220px`、中の `img` は `height: 100%; object-fit: contain` で縦横比を維持
- ロゴ画像が未提供の場合はプレースホルダー(テンプレート内蔵のSVG+社名テキスト)のまま置き、差し替え方法をHTMLコメントで残す
- ダーク背景では白抜きロゴを使う。通常ロゴしかない場合は白の角丸カードを敷くか `filter: brightness(0) invert(1)` で単色白化を検討

## 完成チェックリスト

- [ ] `.slide` が 1280×720px で `overflow: hidden`
- [ ] 1枚で内容が完結(ページ番号・続きを示唆する要素がない)
- [ ] タイトルが結論を言い切るメッセージ文になっている(`<em>` 強調は1箇所)
- [ ] キッカー(左上の英語ラベル)とヘアラインフッターがある
- [ ] 配色がデザインガイドのテーマトークンだけで構成されている(アクセント2色以内)
- [ ] 文字がはみ出し・重なりなく収まり、余白が十分にある
- [ ] スクリーンショットを撮って目視確認した
