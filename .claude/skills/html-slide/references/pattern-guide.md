# パターン選択ガイド(使うべきタイミング一覧)

ユーザーの依頼文・内容から最適なレイアウトパターンを選ぶための判断基準。
**上から順に判定し、最初に条件が合致したパターンを使う。** 迷ったら②ビジネス型。

## 判定フロー

0. **写真・ビジュアルを全面に使いたい**か?(画像提供あり/世界観重視の表紙) → ㉟画像ヒーロー型
1. 資料の**冒頭/末尾の定型パート**か? → 目次なら⑫アジェンダ型、まとめなら㉝まとめ型、行動喚起なら㉚CTA型
2. 依頼に**時系列・順序**があるか? → 手順なら⑥フロー型、四半期/年単位の計画なら⑧ロードマップ型、過去の歩みなら⑱沿革型、詳細工程表なら㉗ガントチャート型、循環するなら㉕サイクル型
3. **数値データ**が主役か? → 推移なら⑩グラフ型、単発実績なら④KPI型、複数指標一覧なら㉘ダッシュボード型、市場規模なら㉑TAM/SAM/SOM型、転換率なら㉓ファネル型
4. **対比・分類**が主題か? → Before/Afterなら⑨対比型、現状→目標+施策なら㉔ギャップ分析型、2軸整理なら⑪マトリクス型、多社の機能比較なら㉒競合比較表型、階層なら㉖ピラミッド型
5. **構造・体制**の説明か? → 組織/体制なら⑭組織図型、技術構成なら⑮システム構成図型、サービス全体像なら⑳ソリューション型
6. **人・実績・信頼**の訴求か? → メンバーなら⑰チーム紹介型、顧客の声なら⑬お客様の声型、取引先実績なら㉜ロゴウォール型、料金なら⑯料金プラン型、FAQなら㉛FAQ型
7. **並列の項目**(特徴・プラン)か? → 3〜4項目なら③3カード型。**種類の違う情報**(価値+KPI+機能など)を1枚で俯瞰させたいなら㉞ベントーグリッド型
8. **メッセージ1つ**を届けたいか? → 課題の問いかけなら⑲課題提起型、名言/理念の言葉なら㉙引用型、表紙/ビジョンなら①コンセプト型(和風なら⑦和モダン型)
9. **toC・カジュアル**な告知か? → ⑤カジュアル型
10. 上記以外(要点説明・提案の骨子) → ②ビジネス型

## 各パターンの使うべきタイミング

| # | パターン | 実例ファイル | 使うべきタイミング(トリガーとなる依頼・内容) | 向かないケース |
|---|---|---|---|---|
| ① | コンセプト型 | slide-refined | 表紙/ビジョン/ブランドコンセプト。「かっこよく」「おしゃれに」「ミニマルに」。テック・クリエイティブ系聴衆 | データ・手順・比較の説明 |
| ② | ビジネス型 | slide-business | 提案書/報告書の要点説明。「箇条書きで」「わかりやすく」。**デフォルト候補** | 感情訴求・toC向け |
| ③ | 3カード型 | slide-cards | 「3つの特徴」「選ばれる理由」「プラン比較」。並列3〜4項目 | 順序のある内容、5項目以上 |
| ④ | KPIデータ型 | slide-kpi | 「実績を大きく見せたい」。売上/導入数/満足度など数値2〜3個の強調 | 推移を見せたい(→⑩)、数値4個以上 |
| ⑤ | カジュアル型 | slide-casual | toCサービス/アプリ紹介、社内イベント告知。「親しみやすく」「ポップに」 | フォーマルな報告・商談 |
| ⑥ | プロセスフロー型 | slide-flow | 「導入の流れ」「手順」「ステップ」。時系列3〜5段階の説明 | 並列項目(→③)、日付ベース計画(→⑧) |
| ⑦ | 和モダン型 | slide-message | 企業理念/周年/伝統・職人系ブランド。「和風で」「格調高く」 | データ説明、グローバル/テック系 |
| ⑧ | ロードマップ型 | slide-roadmap | 「今後の計画」「中期計画」「ロードマップ」。Q1〜Q4等の時間軸+マイルストーン | 手順説明(⑧は「いつ」、⑥は「どうやって」) |
| ⑨ | 対比型 | slide-versus | 「導入前後」「新旧比較」「従来との違い」。2者のコントラスト訴求 | 3者以上の比較(→③)、2軸整理(→⑪) |
| ⑩ | グラフ・チャート型 | slide-chart | 「売上推移」「成長を見せたい」。時系列データ+考察 | 単発数値の強調(→④) |
| ⑪ | マトリクス型 | slide-matrix | 「優先順位を整理」「ポートフォリオ」「ポジショニング」。2軸4象限の分類 | 単純な2者比較(→⑨) |
| ⑫ | アジェンダ型 | slide-agenda | 「目次」「本日の流れ」「議題」。説明会・会議の冒頭 | 内容そのものの説明 |
| ⑬ | お客様の声型 | slide-voice | 「お客様の声」「導入事例」「レビュー」。第三者評価による信頼性補強 | 自社発信の主張・機能説明 |
| ⑭ | 組織図型 | slide-org | 「体制図」「組織図」「チーム構成」。階層関係の提示 | 並列項目(→③)、時系列(→⑥) |
| ⑮ | システム構成図型 | slide-architecture | 「システム構成」「技術スタック」「アーキテクチャ」。IT・エンジニア向け | 非技術者向け概念図(→⑳) |
| ⑯ | 料金プラン型 | slide-pricing | 「料金表」「プラン」「価格」。推しプランを中央強調 | 価格のない比較(→③/㉒) |
| ⑰ | チーム紹介型 | slide-team | 「メンバー紹介」「経営陣」「講師紹介」。人の顔で信頼をつくる | 階層構造(→⑭) |
| ⑱ | 沿革・ヒストリー型 | slide-history | 「沿革」「歩み」「これまでの実績」。過去→現在の物語 | 未来の計画(→⑧) |
| ⑲ | 課題提起型 | slide-problem | 「こんな悩みは」「なぜ〜なのか」。序盤の問題意識喚起 | 解決策の提示(→⑳/⑨) |
| ⑳ | ソリューション全体像型 | slide-solution | 「サービス全体像」「提供価値マップ」。中心+周辺の関係図 | 技術構成(→⑮) |
| ㉑ | 市場規模型 | slide-market | 「市場規模」「TAM/SAM/SOM」「事業機会」。投資家向け | 時系列成長(→⑩) |
| ㉒ | 競合比較表型 | slide-compare-table | 「競合比較」「他社との違い」「機能比較表」。自社列を強調 | 2者の対比(→⑨) |
| ㉓ | ファネル型 | slide-funnel | 「ファネル」「コンバージョン」「集客から成約まで」 | 転換のない手順(→⑥) |
| ㉔ | ギャップ分析型 | slide-gap | 「現状と課題」「あるべき姿」「目標との差」+埋める施策 | 導入前後の結果比較(→⑨) |
| ㉕ | サイクル型 | slide-cycle | 「PDCA」「改善サイクル」「循環」。回り続ける仕組み | 一方向の手順(→⑥) |
| ㉖ | ピラミッド型 | slide-pyramid | 「階層」「土台」「3層構造」。理念→施策などの積み上げ | 並列項目(→③) |
| ㉗ | ガントチャート型 | slide-gantt | 「スケジュール」「工程表」「ガント」。タスク×期間の詳細計画 | 大まかな計画(→⑧) |
| ㉘ | ダッシュボード型 | slide-dashboard | 「月次報告」「実績一覧」「ダッシュボード」。複数指標の一望 | 1数値の強調(→④) |
| ㉙ | 引用・名言型 | slide-quote | 「名言」「創業者の言葉」「キーメッセージ」を1つだけ印象的に | 複数の声(→⑬) |
| ㉚ | CTA・クロージング型 | slide-cta | 「お問い合わせ」「次のステップ」「無料相談」。資料の締めの行動喚起 | 情報提供が目的のページ |
| ㉛ | FAQ型 | slide-faq | 「FAQ」「よくある質問」「想定問答」。不安の払拭 | 1テーマの深い説明(→②) |
| ㉜ | 導入実績ロゴウォール型 | slide-logos | 「導入実績」「取引先一覧」「導入企業」。ロゴを並べて信頼訴求 | 個別事例の深掘り(→⑬) |
| ㉝ | まとめ・チェックリスト型 | slide-summary | 「まとめ」「本日のポイント」。要点再確認+次のアクション | 新情報の提示 |
| ㉞ | ベントーグリッド型 | slide-bento | 「全体像を1枚で」「プロダクト紹介の総括」「月次まとめ」。価値提案+KPI+機能+実績など種類の違う情報の俯瞰 | 1テーマの深掘り(→②)、順序のある内容(→⑥/⑧) |
| ㉟ | 画像ヒーロー型 | slide-hero | 「写真を全面に」「ビジュアル重視で」「世界観を伝えたい」。表紙・ブランド訴求・採用資料の扉。ユーザー提供画像の全面使用 | データ・手順など情報量の多い説明 |

## 組み合わせのヒント

- 各実例ファイルの冒頭コメントにも同じ使用タイミングが書いてある。ベースに流用するときはコメントごと更新する
- どのパターンでも**配色は内容のトーンに合わせて差し替えてよい**(パターン=レイアウト構造の選択であり、色は自由)
- ユーザーが「画像全面」を求めた場合は㉟画像ヒーロー型(slide-hero)をベースに、`.bg-placeholder` を `<img>`(object-fit: cover)に差し替える。オーバーレイ層は必ず残す
